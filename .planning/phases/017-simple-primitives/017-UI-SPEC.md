## Color

| Secondary (30%) | var(--cream-2) | Kbd background (light); Pagination button hover background |

### Per-component color details

**Kbd (light mode)**
- Background: `var(--cream-2)`
- Border: `1px solid var(--rule)` + `box-shadow: 0 1px 0 var(--rule)` (bottom-shadow for keycap depth)
- Text: `var(--ink-2)`
- Border-radius: `4px` (matches `--radius-sm`)

**Kbd (dark mode)**
- Background: `rgba(255, 255, 255, 0.08)` — translucent white over dark surface
- Border: `rgba(255, 255, 255, 0.18)`
- Text: `var(--ink)` (flips to cream-ish in dark token flip)
- Override via `.dark .ds-atom-kbd` CSS rule in primitives.css

**RelativeTime (light + dark mode)**
- Text: `var(--ink-2)` — no dark-mode override needed; token flips automatically
- Prefix span: `var(--ink-4)` — secondary de-emphasized text
- Cursor: `default` (no pointer; informational element)

**Pagination icbtn (prev/next)**
- Idle: `background: var(--g-bg); border: 1px solid var(--rule); color: var(--ink-2)`
- Hover: `background: var(--cream-2); border-color: rgba(0,0,0,0.12)`
- Dark hover: `background: rgba(255,255,255,0.08); border-color: var(--ink-4)`
- Disabled: `opacity: 0.4; cursor: not-allowed`
- Focus-visible: `box-shadow: var(--focus-ring); border-color: var(--focus)`

**Pagination page button**
- Idle: `background: transparent; border: 1px solid transparent; color: var(--ink-2)`
- Hover (inactive): `background: var(--cream-2)`
- Active (aria-current="page"): `background: var(--ink); color: var(--cream); font-weight: 700; cursor: default`
- Focus-visible: `box-shadow: var(--focus-ring); border-color: var(--focus)`

**Pagination ellipsis**
- Text: `var(--ink-4)`
- Non-interactive: `pointer-events: none; user-select: none`

**Pagination label ("Page N of M")**
- Text: `var(--ink-4)`

Source: RESEARCH.md Pattern 1/2/3 CSS blocks; ds-advanced.jsx lines 147–167; ds-universal.jsx lines 208–231.

## Component Inventory

### Kbd

| Property | Value |
|----------|-------|
| HTML element | `<kbd>` |
| CSS class | `ds-atom-kbd` |
| File | `src/inputs/Kbd/index.tsx` |
| Props | `children`, `size?: "sm" \| "md"` (default: `"md"`), extends `HTMLAttributes<HTMLElement>` |
| Ref | `forwardRef<HTMLElement, KbdProps>` |
| Size "sm" | font-size: 9.5px; padding: 1px 5px |
| Size "md" | font-size: 11px; padding: 2px 6px |
| Sequence rendering | Consumer renders multiple `<Kbd>` with `"+"` separators; Kbd renders ONE key only |
| Dark mode | `.dark .ds-atom-kbd` rule in primitives.css (rgba overrides) |
| Accessibility | `<kbd>` native semantic element; no ARIA role needed |

### RelativeTime

| Property | Value |
|----------|-------|
| HTML element | `<time dateTime={isoString}>` |
| CSS class | `ds-atom-relative-time` |
| File | `src/interaction/RelativeTime/index.tsx` |
| Props | `date: Date \| string \| number`, `prefix?: string`, `updateInterval?: number` (default: 60000), extends `HTMLAttributes<HTMLTimeElement>` |
| Ref | `forwardRef<HTMLTimeElement, RelativeTimeProps>` |
| Title attribute | `d.toLocaleString()` — exact datetime on hover |
| dateTime attribute | `d.toISOString()` — machine-readable |
| Live update | `setInterval` in `useEffect`; clears on unmount; re-runs when `date` or `updateInterval` changes |
| Prefix rendering | `<span style={{ color: "var(--ink-4)" }}>{prefix} </span>` — trailing space before relative string |
| Dark mode | No CSS override; `--ink-2` and `--ink-4` flip automatically via token system |
| Accessibility | `<time>` with `dateTime` ISO attribute; `title` tooltip covers hover-readable exact date |

**Format buckets (locked — from REQ-17-02 + handoff ds-dataviz.jsx):**

| Condition | Output |
|-----------|--------|
| `diffMin < 0` (future) | `"in {|diffMin|}m"` |
| `diffMin < 60` | `"{diffMin}m ago"` |
| `diffH < 24` | `"{diffH}h ago"` |
| `diffD < 30` | `"{diffD}d ago"` |
| `diffD >= 30` | `d.toLocaleDateString()` (no "ago") |

Note: `diffMin === 0` produces `"0m ago"` — no special "just now" case per spec.

### Pagination

| Property | Value |
|----------|-------|
| HTML element | `<nav aria-label>` → `<ol>` → `<li>` per item |
| CSS classes | `ds-atom-pagination`, `ds-atom-pagination-list`, `ds-atom-pagination-icbtn`, `ds-atom-pagination-btn`, `ds-atom-pagination-ellipsis`, `ds-atom-pagination-label`, `ds-atom-pagination-count` |
| File | `src/data-display/Pagination/index.tsx` |
| Props | `totalPages: number`, `currentPage: number`, `onPageChange: (page: number) => void`, `variant?: "full" \| "compact"` (default: `"full"`), `ariaLabel?: string` (default: `"Pagination"`), `className?`, `style?` |
| Ref | `forwardRef<HTMLElement, PaginationProps>` |
| Controlled | Yes — caller owns `currentPage` state |
| Variant: full | prev icbtn + numbered page buttons + ellipsis + next icbtn + "Page N of M" label |
| Variant: compact | prev icbtn + "N / M" (mono 11px 600) + next icbtn |
| Dark mode | `.dark .ds-atom-pagination-icbtn:hover` rule in primitives.css |

**Pagination full — button dimensions:**
- `ds-atom-pagination-icbtn`: 28×28px, border-radius: 7px
- `ds-atom-pagination-btn`: 28×28px, border-radius: 7px
- `ds-atom-pagination-ellipsis`: 28×28px (non-interactive, display only)

**Page range algorithm (getPageRange):**
- If `totalPages <= 7`: show all pages — no ellipsis
- Otherwise: always show page 1 and page `totalPages`; show `currentPage ± 1`; insert `"…"` where gaps exist
- Ellipsis rendered as `<li aria-hidden="true"><span className="ds-atom-pagination-ellipsis" aria-hidden="true">…</span></li>`

**Active page:**
- `aria-current="page"` attribute on active button
- CSS selector: `.ds-atom-pagination-btn[aria-current="page"]` → ink background, cream text

**Keyboard navigation:**
- All page buttons receive `tabIndex={0}` (simple approach — acceptable for phase scope per RESEARCH.md)
- `ArrowLeft`/`ArrowRight` on the `<ol>` `onKeyDown` moves `focus()` to previous/next sibling button
- `Enter` on a focused button fires `onPageChange`

**Accessibility:**
- `<nav aria-label="Pagination">` landmark
- `aria-label="Previous page"` / `"Next page"` on icbtns
- `aria-label="Page N"` on each numbered button
- `aria-hidden="true"` on ellipsis `<li>` and `<span>`
- `disabled` attribute on prev when `currentPage <= 1`; next when `currentPage >= totalPages`

## Copywriting Contract

Phase 17 contains no user-facing copy beyond data-driven strings (dates, page numbers). There is no primary CTA, empty state, error state, or destructive action.

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — no user-initiated actions in Kbd or RelativeTime |
| Empty state | Not applicable — Pagination with `totalPages: 0` renders nothing (consumer guards this) |
| Error state | Not applicable — no network calls or async operations |
| Destructive confirmation | Not applicable — no destructive actions |

**Pagination prev/next accessible labels (locked):**
- Previous button: `aria-label="Previous page"`
- Next button: `aria-label="Next page"`
- Each numbered button: `aria-label="Page {N}"`
- Nav landmark: `aria-label="Pagination"` (default; overridable via `ariaLabel` prop)

**RelativeTime tooltip:**
- `title` attribute value: `d.toLocaleString()` — browser locale; no override needed
